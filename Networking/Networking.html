  • A Secure Sockets Layer certificate is a digital file that confirms the identity of a website and encrypts the data that is sent between the website and your browser.
  • When you visit a website that has an SSL certificate, the address bar in your browser will show a green padlock or the word "Secure". This means that the website is using an SSL certificate and that your data is secure.
  • You can get an SSL certificate from a number of different providers, such as Comodo, GoDaddy, and Let's Encrypt.
  • An SSL handshake is a process that takes place between a web browser and a web server when a user visits a website that uses HTTPS (HTTP over SSL/TLS) :
    • The browser sends a message to the server requesting a secure connection. ->
    • The server sends its SSL certificate to the browser. ->
    • The browser verifies the certificate (when a web browser connects to a website that uses HTTPS, the browser will compare the certificate that it receives from the server with the certificates that are stored in the Windows Certificate Store. If the certificates match, the browser will trust the website and allow the connection to proceed.) and creates a session key. ->
    • The browser and server use the session key to encrypt all subsequent communication.

SOCKS (Socket Secure, SOCKS4 and SOCKS5) is a networking protocol. It is like a secret tunnel that you can use to go online.

  1. The client sends a request to the SOCKS proxy.
  2. The SOCKS proxy authenticates the client.
  3. The SOCKS proxy establishes a connection to the server.
  4. The SOCKS proxy tunnels all traffic between the client and the server.
  5. The SOCKS proxy terminates the connection to the server.

It is a popular choice for tunneling traffic over a VPN, for bypassing firewalls, and for hiding the user's IP address.

  1. HTTPS is more widely supported than SOCKS. Most websites and applications support HTTPS, while only a few support SOCKS.
  2. HTTPS is simpler to set up and use than SOCKS. Most websites and applications automatically use HTTPS, while SOCKS requires manual configuration.
  3. If you use a VPN with SOCKS and still access a website that uses HTTPS, your traffic will be encrypted twice. The first layer of encryption will be provided by the VPN, and the second layer of encryption will be provided by HTTPS.
  4. If you are only using SOCKS to access a regular website that uses HTTPS, your traffic will be encrypted by the HTTPS protocol. However, your IP address will still be visible to the website. This means that the website will be able to see where you are connecting from.

It is an architectural style for designing web services. RESTful web services use HTTP requests to access and manipulate resources.

REST is used to create secure and efficient web services.

  • 1. Resource-oriented: RESTful web services are based on the concept of resources. A resource is anything that can be identified and manipulated. For example, a resource could be a product, a customer, or an order.
  • 2. Stateless: each request is independent of the previous request. The server does not need to keep track of the state of the client.
  • 3. Uniform Interface: all requests use the same HTTP methods and the same resource identifiers.
  • 4. Caching: can improve performance by reducing the number of requests that need to be made to the server.
